# Contributing to PunamIDE

Thank you for helping test PunamIDE.

PunamIDE source code is private during early alpha, so this public repository is focused on:

- Bug reports.
- Feature requests.
- Alpha feedback.
- Release discussion.
- Community support.

## Reporting Bugs

Use GitHub Issues and choose the **Bug Report** template.

Helpful bug reports include:

- PunamIDE version.
- Windows version.
- Steps to reproduce.
- Expected behavior.
- Actual behavior.
- Screenshots or short videos.
- Logs or diagnostics exported from PunamIDE.

Do not include API keys, tokens, private source code, or sensitive project data.

## Requesting Features

Use GitHub Issues or GitHub Discussions and describe:

- The problem you are trying to solve.
- The workflow you expected.
- Why it matters.
- Any alternatives you tried.

Feature requests are especially useful when they describe a real development workflow.

## Joining Alpha Testing

Join the Discord:

[https://discord.gg/PFp9KWY3eY](https://discord.gg/PFp9KWY3eY)

Use the Discord channels for:

- Release announcements.
- Download questions.
- Bug reports.
- Feature requests.
- General discussion.

## Discord Workflow

Recommended flow:

1. Check `#announcements` for the latest alpha release.
2. Download the latest installer from GitHub Releases.
3. Report bugs in GitHub Issues or `#bug-reports`.
4. Share feature ideas in GitHub Discussions or `#feature-requests`.
5. Include diagnostics when possible.

## Source Code Contributions

Source code contributions are not open during early alpha.

As PunamIDE matures, public contribution options may expand.
